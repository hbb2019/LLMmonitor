## 八、推荐监控指标
本章的目标不是把所有可记录的数值都纳入日志，而是建立一套**可定位、可判断、可触发保护、可复盘**的稳定性监控体系。监控指标应服务于全文前述闭环：发现异常、判断异常属于根因还是症状、定位异常传播路径、决定是否需要 A 类根因修复或 B/C 类数值保护机制。

本章延续全文中对 loss、logits、attention scores、hidden states / residual stream、activation、gradient、parameter / weight、optimizer state、optimizer update、learning rate / schedule、batch data、sequence / mask、precision、distributed consistency、MoE routing 等对象的分类要求。

---

### 8.1 监控指标设计原则
稳定性监控不应只记录标量均值。大模型训练中的很多异常首先表现为**局部分布变形**，例如某一层 residual RMS 上升、某几个 attention head entropy collapse、某个 expert load 过载、某个 rank overflow，平均值可能在早期完全正常。

推荐每个重要对象至少记录以下五类统计：

| 统计类型 | 推荐统计量 | 用途 | 典型问题 |
| --- | --- | --- | --- |
| 中心趋势 | mean、median、RMS | 判断整体尺度是否漂移 | residual stream 整体变大、logits 整体变尖 |
| 尾部风险 | max、p95、p99、p999 | 捕捉 outlier 和爆炸前兆 | activation outlier、attention score spike |
| 离散程度 | std、IQR、CV | 判断分布是否变宽或专家负载是否不均 | MoE expert imbalance、layer-wise norm 分化 |
| 形状信息 | histogram、kurtosis、skewness | 判断分布是否重尾、截断、饱和 | clip 后分布硬截断、logits 饱和 |
| 非有限值与异常比例 | NaN ratio、Inf ratio、outlier ratio、clip rate、overflow count | 判断是否进入危险状态 | fp16 overflow、gradient explosion、loss scaler 频繁下降 |


其中，**p99 / p999、histogram、outlier ratio、clip rate** 通常比 mean 更有诊断价值。只看均值容易漏掉以下问题：

| 只看均值可能漏掉的问题 | 更合适的监控方式 |
| --- | --- |
| 少数 token logits 极端 | logits max、p99、top-token frequency |
| 少数 sequence loss 极端 | per-sequence loss top-k、p99 |
| 少数 attention head collapse | per-head entropy histogram |
| 少数 expert 过载 | expert load CV、max load / mean load |
| 少数 rank overflow | per-rank overflow count、loss scale |
| 少数层 update 过大 | layer-wise update-to-weight ratio p99 |


---

### 8.2 指标分层：从全局到局部
推荐按以下层级组织监控，而不是把指标平铺：

| 层级 | 作用 | 典型问题 | 主要用于 |
| --- | --- | --- | --- |
| global 级别 | 判断训练是否整体偏离稳定区间 | loss spike、global grad explosion、LR/resume 错误 | 异常发现、初步分流 |
| layer 级别 | 判断异常从哪一层开始传播 | residual RMS drift、某层 grad 爆 | 根因定位 |
| attention head 级别 | 判断 attention score、mask、Q/K、entropy 问题 | attention entropy collapse、非法 attention | 根因定位 |
| MLP / FFN 级别 | 判断 gate/up/down activation outlier | MLP 输出尺度异常、SwiGLU outlier | 根因定位 |
| embedding / lm head 级别 | 判断 vocab、tokenizer、logits、输出层尺度问题 | logits explosion、特殊 token 异常 | 根因定位 |
| optimizer state 级别 | 判断 Adam m/v、state、epsilon、resume 是否异常 | update 大但 grad 不大 | 根因定位 |
| optimizer update 级别 | 判断实际参数更新是否失控 | update-to-weight ratio 异常 | 根因定位 + 保护触发 |
| data batch 级别 | 判断 batch、sample、sequence、mask、label 是否异常 | 特定 batch 复现 spike | 根因定位 |
| distributed rank 级别 | 判断多卡、ZeRO/FSDP、loss scale 是否一致 | 单 rank NaN、多卡不一致 | 根因定位 |
| MoE expert 级别 | 判断 router、expert load、capacity 是否稳定 | expert collapse、router logits 爆 | 根因定位 + 保护触发 |


工程上建议采用两级采样频率：

| 监控频率 | 指标类型 | 推荐用途 |
| --- | --- | --- |
| 每 step 或每 few steps | loss、LR、global grad norm、update norm、loss scale、overflow、clip rate、tokens/update | 快速发现异常 |
| 每 N step 或异常时触发 | layer-wise histogram、attention entropy、per-token loss、per-rank checksum、expert load histogram | 精细定位根因 |
| 异常 replay 时 | full hook、per-layer forward/backward tensor stats、sample-level loss、mask legality check | 复盘与最小验证 |


---

### 8.3 global 级别指标
global 指标用于判断训练是否进入异常状态，并为后续定位选择方向。它们不应该直接替代 layer-wise 或 data-wise 诊断，因为 global 指标通常只能说明“出问题了”，不能说明“为什么出问题”。

| 监控对象 | 推荐统计量 | 异常信号 | 可能根因 | 推荐处理方式 | 更适合用于 |
| --- | --- | --- | --- | --- | --- |
| training loss | mean、EMA、p95、p99、spike ratio、Δloss/step | loss 突然升高；连续多个 step 上升；p99 明显高于历史区间 | 异常 batch、LR 过大、optimizer state 错误、precision overflow、mask/label bug | 保存 bad step checkpoint 和 batch；replay 同一 batch；查看 per-sequence loss 与 update norm | 异常发现 + 根因定位入口 |
| validation loss | mean、trend、domain-wise loss | train loss 稳定但 valid loss 持续恶化；某 domain valid loss 突变 | 数据分布偏移、过拟合、clip 过强、训练目标错误 | 按数据域拆分验证；检查数据混合比例；检查 clip rate 与 valid loss 相关性 | 根因定位 |
| global grad norm | norm、EMA、max、p95、non-finite ratio | 突然放大；出现 NaN/Inf；频繁触发 grad clipping | loss spike、activation explosion、LR 过大、异常 batch | 切到 layer-wise grad norm；检查 loss 是否先 spike；必要时触发 global grad clipping | 保护触发 + 定位入口 |
| global update norm | norm、EMA、p95、max | grad norm 不大但 update norm 大；resume 后 update 突变 | Adam v 过小、epsilon 配置、optimizer state 损坏、LR group 错误 | 检查 optimizer state、param group LR、update-to-weight ratio；对比 reset optimizer replay | 根因定位 |
| update-to-weight ratio | mean、p95、p99、max、by-param-group | 某些层或参数组 U/W 远高于其他层；训练质量下降但 grad norm 正常 | LR 配错、weight norm 过小、Adam state 异常、decay 配置错误 | 分层记录 U/W；必要时触发 update cap；优先检查 optimizer state 和 LR group | 根因定位 + 保护触发 |
| learning rate | current LR、by param group、scheduler step、warmup progress | resume 后 LR 不连续；某 param group LR 异常；warmup 未生效 | scheduler state 错误、global step 错、param group 配错 | 打印并记录每个 param group；与 checkpoint metadata 对齐 | 根因定位 |
| effective batch size | samples/update、tokens/update、microbatch 数、gradient accumulation step | 实际 tokens/update 与配置不一致；loss scale 与 batch 改变不同步 | dataloader、packing、accumulation、world size 变化 | 记录每 step tokens；核对 LR scaling；验证 accumulation 等价性 | 根因定位 |
| precision / loss scale | loss scale、overflow count、skipped step count、found_inf | loss scale 频繁下降；连续 skip step；某 step overflow 后 loss spike | fp16 overflow、kernel 数值不稳定、activation/logits 过大 | bf16/fp32 单步 replay；定位第一个 non-finite tensor；关闭可疑 fused kernel | 根因定位 |
| clip rate | global clip rate、layer clip rate、update clip rate、softcap saturation rate | clip 长期高频触发；clip 后 loss 稳但 valid 恶化 | clip 掩盖 LR、数据、架构或 optimizer 根因 | 记录 clip 前后 norm 和方向 cosine；降低 LR 或修数据后比较 clip rate | 判断保护是否掩盖根因 |
| throughput / step time | tokens/sec、step time、data time、communication time | step time 突变；某些 step 慢且 loss spike | 极长样本、packing 异常、通信阻塞、rank 数据不均 | 关联 seq length、rank batch shape、expert load | 辅助定位 |
| global non-finite count | NaN count、Inf count、first non-finite step | 出现任何非有限值 | precision、softmax/log/exp、除零、kernel bug | 立即保存现场；开启 anomaly hook；replay 定位第一处 non-finite | 异常发现 |


判断标准建议：

| 指标现象 | 优先排查方向 |
| --- | --- |
| loss spike 先出现，grad norm 后出现 | 数据、label、mask、logits、loss reduction |
| grad norm 先升高，loss 后升高 | activation、residual、attention/MLP、LR |
| grad norm 正常但 update norm 大 | optimizer state、Adam v、epsilon、LR group |
| resume 后第一步异常 | scheduler、optimizer state、loss scaler、ZeRO/FSDP state |
| fp16 overflow 频繁但 bf16/fp32 正常 | precision range、loss scaling、kernel 数值稳定性 |
| global 指标正常但 valid loss 退化 | 目标函数、数据混合、clip 过强、过拟合 |

### 8.4 layer 级别指标
layer 级别指标用于回答：**异常从哪一层开始出现，是逐层累积还是单层跳变，是 forward 激活问题还是 backward 梯度问题。**

| 监控对象 | 推荐统计量 | 异常信号 | 可能根因 | 推荐处理方式 | 更适合用于 |
| --- | --- | --- | --- | --- | --- |
| layer input hidden states | RMS、mean、std、p95、p99、max、histogram | 某层输入 RMS 相比前一层突然放大；深层 RMS 单调上升 | residual stream 累积、前序层 attention/MLP 输出过大 | 定位第一个异常层；拆分 attention output 与 MLP output | 根因定位 |
| layer output hidden states | RMS、p99、max、outlier ratio | 输出/input ratio 异常；某层输出分布重尾 | block 内 attention 或 MLP 尺度异常 | 记录 block output/input ratio；检查该层参数和梯度 | 根因定位 |
| residual branch delta | ||block output - input||、delta/input ratio | residual 增量持续大于输入尺度；深层累积放大 | residual scaling 不合理、LR 过大、初始化问题 | 调 residual scaling、检查 Pre-LN/Post-LN、降低 LR replay | 根因定位 |
| layer grad norm | norm、p95、max、non-finite ratio | 某层 grad norm 远高于相邻层；出现 NaN/Inf | 局部模块异常、loss spike 后传、mask/attention 问题 | backward hook 定位；必要时 per-layer grad clipping 止血 | 定位 + 保护触发 |
| layer param norm | norm、RMS、max、histogram | 某层权重范数持续漂移；norm 参数异常大 | weight decay 配置、LR group、optimizer state | 检查 param group；比较 weight decay contribution | 根因定位 |
| layer update norm | norm、p95、max | 某层 update 突然升高；grad norm 不对应 | Adam v、epsilon、LR group、state shard 错误 | 查看 m/v 与 update-to-weight ratio | 根因定位 |
| layer update-to-weight ratio | mean、p95、p99、max | U/W 长期高于其他层；某层反复触发 update cap | LR group 错、weight norm 小、optimizer state 异常 | 对该层做 LR 降低或 state reset 对比；必要时 U/W cap | 定位 + 保护触发 |
| norm layer input/output | input RMS、output RMS、gamma norm、beta norm | LN/RMSNorm input RMS 异常但 output 被压平；gamma 异常增长 | 上游尺度漂移被 norm 掩盖、训练动力学异常 | 不只看 norm 后输出，要监控 norm 前输入；检查 gamma 变化 | 判断是否被归一化掩盖 |
| non-finite by layer | NaN/Inf count、first non-finite layer | 某层首次出现非有限值 | softmax、MLP activation、division/sqrt、kernel bug | 定位 first bad tensor；fp32 replay；关闭 fused op | 根因定位 |


推荐额外记录两个派生指标：

[  
\text{block amplification ratio}_l = \frac{|h_{l+1}|_{\mathrm{RMS}}}{|h_l|_{\mathrm{RMS}}}  
]

[  
\text{residual delta ratio}_l = \frac{|h_{l+1}-h_l|_{\mathrm{RMS}}}{|h_l|_{\mathrm{RMS}}+\epsilon}  
]

判断标准：

| 现象 | 更可能的解释 |
| --- | --- |
| 所有层 residual RMS 随深度缓慢上升 | residual scaling、初始化、LR、Pre-LN/Post-LN recipe 问题 |
| 某一层 output/input ratio 阶跃上升 | 该层 attention 或 MLP 局部问题 |
| norm 前 RMS 异常但 norm 后正常 | LayerNorm/RMSNorm 正在掩盖上游尺度漂移 |
| backward 某层 grad 爆但 forward 激活正常 | loss/logits、optimizer、梯度路径或局部 Jacobian 问题 |
| forward 某层 activation 先爆，随后 grad 爆 | 该层或上游层是优先排查对象 |

### 8.5 attention head 级别指标
attention 监控的核心不是只看 attention output norm，而是同时看 **Q/K norm、QK score、mask 合法性、softmax entropy、max attention probability**。很多 attention 问题会先表现为 entropy collapse 或非法 attention，而不是 loss 立即爆炸。

| 监控对象 | 推荐统计量 | 异常信号 | 可能根因 | 推荐处理方式 | 更适合用于 |
| --- | --- | --- | --- | --- | --- |
| Q norm | per-head mean、RMS、p95、p99、max | Q norm 持续增大；某 head Q norm 远高于其他 head | Wq scale、LR、QK score 尺度问题 | 检查 Q/K norm 与 score 相关性；考虑 QK Norm 或降低 LR | 根因定位 |
| K norm | per-head mean、RMS、p95、p99、max | K norm 持续增大；长序列时更明显 | Wk scale、RoPE scaling、position id 问题 | 按 sequence length 分桶；检查 RoPE/position id | 根因定位 |
| V norm | RMS、p99、max | V 正常但 output 异常，可能是 attention weight 问题；V 本身异常则查 projection | Wv scale、上游 hidden 异常 | 与 attention weight entropy 联合判断 | 根因定位 |
| QK scores / attention logits | p95、p99、max、histogram | score 极端大；分布重尾；softmax 前已有 Inf | Q/K norm 过大、temperature 错、RoPE 问题、mask 加法错误 | 记录 score before/after mask；检查 mask value；考虑 score softcap/QK Norm | 定位 + 保护触发 |
| attention mask legality | illegal attention count、future attention ratio、cross-sample attention ratio、pad attention ratio | attend 到未来 token、padding token 或 packed boundary 外样本 | causal mask、padding mask、packing mask、position id bug | 构造 toy batch 单元测试；记录非法 attention 热点 | 根因定位 |
| attention entropy | per-head mean、min、p5、histogram | entropy collapse 接近 one-hot；或 entropy 过高接近均匀 | score 尺度过大/过小、mask 错、temperature 不当 | 对比 score p99；检查 Q/K norm；调 temperature 或 QK Norm | 根因定位 |
| max attention probability | per-head p95、p99、max | 某些 head 长期 max prob 接近 1 | attention 过尖、score 饱和、mask 错 | 检查这些 head 的 QK score 和 token position | 根因定位 |
| attention output norm | RMS、p99、max | attention output 放大 residual stream | attention weight 饱和、V norm 过大、output proj scale | 分解 V norm、attention weight、out_proj | 根因定位 |
| head-wise grad norm | norm、p95、max | 少数 head 梯度异常 | 局部 head collapse、mask 或数据异常 | head-wise backward hook；必要时局部 clip | 定位 + 保护触发 |
| attention dropout / mode | dropout rate、train/eval flag | eval/train 模式错误导致行为异常 | mode 切换 bug | 检查 model.training 与 dropout config | 根因定位 |


典型判断：

| 指标组合 | 优先判断 |
| --- | --- |
| Q/K norm 大，QK score 大，entropy collapse | attention score 尺度问题，考虑 QK Norm、temperature、LR、RoPE |
| Q/K norm 正常，但 mask 后 score 异常 | mask value、mask dtype、causal/padding/packing mask bug |
| entropy 正常但 attention output norm 大 | V projection 或 out_proj 尺度问题 |
| 长序列才出现 score 爆 | RoPE scaling、position id、long-context recipe 或 length curriculum 问题 |
| 某些 head 长期 one-hot 且 loss 变差 | attention collapse 可能已影响学习，需要排查而不是只 clamp score |

### 8.6 MLP / FFN 级别指标
MLP / FFN 通常占 Transformer 参数和计算的大头，SwiGLU、GeGLU、GELU 等激活函数会产生局部 outlier。MLP 问题常见表现是：attention 看起来正常，但 FFN 输出使 residual stream 放大。

| 监控对象 | 推荐统计量 | 异常信号 | 可能根因 | 推荐处理方式 | 更适合用于 |
| --- | --- | --- | --- | --- | --- |
| gate projection output | RMS、p95、p99、max、kurtosis | gate outlier；分布极重尾；个别 token gate 极大 | SwiGLU/GeGLU scale、初始化、LR、输入 hidden outlier | 检查 activation 前后分布；按 token 定位 outlier | 根因定位 |
| up projection output | RMS、p99、max | up_proj 激活持续变大 | fan-in 初始化、hidden RMS 上游异常 | 检查 up_proj weight norm 与输入 hidden RMS | 根因定位 |
| activation after nonlinearity | RMS、p99、max、saturation ratio | GELU/SwiGLU 后数值极大或大量饱和 | 激活函数实现、dtype、输入过大 | fp32 replay；检查 fused activation kernel | 根因定位 |
| elementwise gated product | RMS、p99、max | gate × up 后突然重尾 | gate/up 同时 outlier，SwiGLU 乘性放大 | 记录 gate、up、product 三者 histogram | 根因定位 |
| down projection output | RMS、p99、max、delta/input ratio | FFN 输出使 residual delta 过大 | down_proj scale、MLP hidden outlier | 检查 down_proj weight norm；必要时 residual scaling | 根因定位 |
| MLP grad norm | norm、p95、max | FFN 参数梯度异常高 | activation outlier、loss spike、LR | 与 forward activation outlier 对齐；必要时 layer-wise clipping | 定位 + 保护触发 |
| MLP update-to-weight ratio | mean、p95、max | MLP 层 U/W 高于 attention 层很多 | LR group、Adam state、FFN 参数尺度 | 检查 param group 和 optimizer state | 根因定位 |
| activation outlier ratio | 超过阈值比例、p99/median ratio | p99 与 median 脱钩；局部 token 激活极大 | heavy-tail activation、异常 token、precision | 定位 token/sample；检查是否与特定数据源相关 | 根因定位 |


典型判断：

| 指标组合 | 优先判断 |
| --- | --- |
| gate/up 正常，product 异常 | 乘性 gating 放大，重点检查 SwiGLU/GeGLU scale |
| MLP output 异常但 attention output 正常 | FFN 是 residual 放大的主要来源 |
| activation outlier 与特定 token/source 绑定 | 数据或 tokenizer/template 问题 |
| fp32 正常、bf16/fp16 异常 | fused MLP kernel 或低精度 overflow |
| down_proj output/input ratio 长期过大 | residual branch scale、MLP 初始化或 LR 需要修正 |

### 8.7 embedding / lm head 级别指标
embedding 和 lm head 是 logits、tokenizer、vocab、special token 问题最容易暴露的位置。尤其在 tokenizer 变更、vocab resize、tie embedding、SFT 模板、RLHF 数据格式变化时，这一层级必须单独监控。

| 监控对象 | 推荐统计量 | 异常信号 | 可能根因 | 推荐处理方式 | 更适合用于 |
| --- | --- | --- | --- | --- | --- |
| token embedding row norm | mean、p95、p99、max、by-token top-k | 特殊 token、pad/eos/bos、unused token row norm 异常 | tokenizer/template、vocab resize、LR group 错误 | 按 token id 记录 row norm top-k；检查 special ids | 根因定位 |
| position embedding / RoPE input | position id min/max、range、重复率 | position id 越界、重置错误、packed sequence 中错位 | position id bug、packing bug、long-context scaling 问题 | 构造 toy sequence 检查 position id | 根因定位 |
| final hidden RMS | RMS、p99、max | final hidden 已很大，导致 logits 大 | residual stream 上游问题 | 回溯 layer-wise RMS | 根因定位 |
| lm head weight norm | row norm mean、p99、max | 某些 vocab row 异常大；logits top token 固定异常 | weight tying、LR group、vocab mismatch | 检查 tied weight；查看 top logit token | 根因定位 |
| output logits | RMS、max、p95、p99、entropy、top-1 prob | logits 极端；softmax entropy collapse；top-1 prob 过高 | final hidden 大、lm_head scale、tokenizer/label 问题 | 检查 final hidden 与 lm_head norm；必要时 logit softcap | 定位 + 保护触发 |
| vocab top-k distribution | top logit token frequency、target token rank | 某些 token 长期占据 top logit；特殊 token 异常高 | tokenizer/template、label shift、数据格式 bug | 按 token id 和样本源分析；检查 label shift | 根因定位 |
| label distribution | target token histogram、ignore ratio、special token ratio | label 中 pad/eos/system token 比例异常 | SFT/RLHF template、ignore_index 错 | 检查 loss mask；按 role 或 segment 拆分 | 根因定位 |
| tied embedding consistency | checksum、pointer identity、weight diff norm | supposed tied weights 实际未 tied；更新不同步 | weight tying 实现 bug、checkpoint load 错 | 检查参数引用和 state_dict | 根因定位 |


典型判断：

| 现象 | 优先排查方向 |
| --- | --- |
| logits 极端但 final hidden RMS 正常 | lm head row norm、vocab、weight tying、tokenizer |
| final hidden RMS 已异常 | 回到 layer-wise residual / attention / MLP |
| 特殊 token loss 或 logit 异常 | template、ignore_index、label shift |
| vocab resize 后训练不稳 | embedding 初始化、optimizer state、tied weight、special id |
| SFT 中 loss 异常低或异常高 | assistant-only loss mask、label shift、prompt token 是否参与 loss |


---

### 8.8 optimizer state 级别指标
optimizer state 级别监控用于区分“梯度本身大”和“优化器把正常梯度放大成异常 update”。对于 Adam 类优化器，必须监控 m、v、effective update，而不仅是 gradient norm。

| 监控对象 | 推荐统计量 | 异常信号 | 可能根因 | 推荐处理方式 | 更适合用于 |
| --- | --- | --- | --- | --- | --- |
| Adam first moment m | norm、RMS、p95、max、finite ratio | m 突变；m 出现 NaN/Inf；resume 后 m 与 grad 不连续 | optimizer state 损坏、resume 错误、梯度异常 | 检查 state_dict；与连续训练 checkpoint 对比 | 根因定位 |
| Adam second moment v | min、mean、p95、p99、max、finite ratio | v 过小导致 effective update 大；v NaN/Inf | epsilon 太小、state 丢失、bias correction 错、稀疏梯度 | 记录 (m/\sqrt v)；必要时调 epsilon 或恢复 state | 根因定位 |
| bias correction step | optimizer step、state step、global step | step 归零或不连续；resume 后 bias correction 异常 | optimizer step 未加载、scheduler/optimizer step 不一致 | 对齐 global step、optimizer step、scheduler step | 根因定位 |
| effective Adam update | (m/(\sqrt v+\epsilon)) norm、p95、max | grad norm 正常但 effective update 很大 | v 小、epsilon、state 错、稀疏参数 | 分参数组定位；检查 v 分布 | 根因定位 |
| weight decay contribution | decay update norm、decay/update ratio | decay 占比异常高；norm/bias 被 decay | weight decay 配错、AdamW param group 错误 | 分解 gradient update 与 decay update | 根因定位 |
| optimizer state finite ratio | finite ratio、NaN/Inf count | state 中已有非有限值 | checkpoint 损坏、overflow 扩散、通信错误 | 回滚 checkpoint；修 ZeRO/FSDP state load | 根因定位 |
| state shape / dtype | dtype、shape、device、shard metadata | state shape 不匹配；dtype 错误；shard 错位 | checkpoint resume、ZeRO/FSDP 重分片错误 | 校验 state metadata 和参数映射 | 根因定位 |
| optimizer state memory | allocated memory、state shard size | 某 rank state size 异常 | shard mapping、param group 不一致 | rank-wise state size 对比 | 分布式根因定位 |


推荐记录的派生指标：

[  
\text{effective update norm}_l = \left|\frac{m_l}{\sqrt{v_l}+\epsilon}\right|  
]

[  
\text{decay ratio}_l = \frac{|\lambda W_l|}{\left|\frac{m_l}{\sqrt{v_l}+\epsilon}\right|+\epsilon}  
]

判断标准：

| 指标组合 | 优先判断 |
| --- | --- |
| grad norm 正常，effective update 大 | Adam v / epsilon / state 问题 |
| resume 后 m/v 分布突变 | optimizer state 加载错误 |
| decay ratio 异常高 | weight decay 配置或参数组问题 |
| 某些参数 v 长期接近 0 | 稀疏更新、冻结/解冻、epsilon 需要检查 |
| state 中出现 NaN/Inf | 不能只 skip step，应检查 state 是否污染并回滚 |

### 8.9 optimizer update 级别指标
optimizer update 是大模型稳定性监控中最容易被忽视但最关键的一层。训练真正改变模型的是 (\Delta W)，不是裸梯度 (g)。很多情况下，gradient 看起来正常，但经过 Adam、weight decay、loss scaling、param group LR、ZeRO/FSDP state 后，实际 update 已经失控。

| 监控对象 | 推荐统计量 | 异常信号 | 可能根因 | 推荐处理方式 | 更适合用于 |
| --- | --- | --- | --- | --- | --- |
| raw gradient update component | norm、p95、max | 梯度贡献异常大 | loss spike、activation explosion | 回到 grad/layer/data 诊断 | 根因定位 |
| Adam-adapted update | norm、p95、max、histogram | adaptive update 与 grad norm 不匹配 | v 过小、epsilon、state 错误 | 检查 optimizer state | 根因定位 |
| weight decay update component | norm、ratio to total update | decay 贡献过大 | WD 配错、norm/bias 被 decay、LR group 错 | 分解 update；修 param group | 根因定位 |
| total parameter update (\Delta W) | norm、RMS、p95、p99、max | 某层或全局 update 突然增大 | LR、optimizer state、grad spike | 记录 by-layer update；必要时 update clipping | 定位 + 保护触发 |
| update-to-weight ratio | mean、p95、p99、max、histogram | U/W 远超历史区间；小权重层比例极大 | LR group、state、weight norm、初始化 | 先查 LR/state；必要时 U/W cap | 根因定位 + 保护触发 |
| update direction cosine | cosine(update, grad)、cosine(update before/after clipping) | clip 或 adaptive update 后方向变化大 | clipping 过强、Adam state 异常 | 监控 clip 是否重写方向；降低 clip 频率 | 判断保护副作用 |
| clipped update ratio | 被 clip 参数比例、clip 前后 norm ratio | update clipping 高频触发 | 根因未修、LR 过大、state 异常 | 若持续触发，回到 A 类排查 | 判断是否掩盖根因 |
| trust ratio | (|W| / |\Delta W|) 或反比 | 层间 trust ratio 差异极大 | 参数尺度不均、LR group 问题 | layer-wise LR 或 U/W cap | 根因定位 |


关键判断：

| 现象 | 说明 |
| --- | --- |
| grad norm 大，update norm 也大 | 可能是真正梯度爆炸或异常 batch |
| grad norm 正常，update norm 大 | 优先查 Adam state、v、epsilon、LR group |
| update-to-weight ratio 只在 embedding/lm_head 异常 | 查 tokenizer/vocab、embedding LR、weight tying、row norm |
| update clipping 长期触发 | 保护机制正在参与定义训练动力学，不应只当作保险丝 |
| clip 前后 direction cosine 很低 | clip 已显著改变优化方向，需要评估质量副作用 |

### 8.10 data batch 级别指标
data batch 级别监控用于判断异常是否由特定 batch、sample、source、sequence length、mask、label 或 sample weight 触发。对于 loss spike，data batch 级别通常是最高优先级排查对象之一。

| 监控对象 | 推荐统计量 | 异常信号 | 可能根因 | 推荐处理方式 | 更适合用于 |
| --- | --- | --- | --- | --- | --- |
| sequence length | mean、p95、p99、max、histogram | bad step 中 length p99/max 异常；长序列集中触发 spike | 极长样本、length curriculum、packing 问题 | 按长度分桶 replay；限制极端长度或调整 curriculum | 根因定位 |
| tokens per sample / per batch | total tokens、non-pad tokens、effective loss tokens | tokens/update 与配置不一致；loss token 数异常少或异常多 | packing、padding、loss mask bug | 记录 effective tokens；核对 loss reduction | 根因定位 |
| per-sequence loss | mean、p95、p99、top-k sample ids | 少数 sequence loss 极端；top-k 样本稳定触发 spike | 脏数据、错误 label、模板错误、异常语言/格式 | 保存 top-k sample；单样本/小组 replay | 根因定位 |
| per-token loss | p95、p99、top token ids、by-position loss | 某些 token 或 position loss 极端 | tokenizer、label shift、special token、position/mask | 按 token id、position、segment 拆分 | 根因定位 |
| data source / domain | source ratio、domain-wise loss | 某 source loss spike；采样比例突变 | sampler bug、数据混合漂移、低质量源 | source-wise replay；调整采样或清洗 | 根因定位 |
| language / script ratio | language histogram、unicode category | 某语种/脚本突然占比异常 | 数据管道、过滤器、采样错误 | 按语言分桶 loss | 根因定位 |
| special token ratio | bos/eos/pad/unk/system/user/assistant token ratio | 特殊 token 比例异常；pad token 参与 loss | tokenizer/template/ignore_index 错 | 检查模板和 loss mask | 根因定位 |
| label ignore ratio | ignore_index ratio、valid label ratio | valid label 过少或过多；padding 未 ignore | loss mask 错、SFT assistant-only mask 错 | 手算一个 batch loss；tiny batch overfit | 根因定位 |
| sample weight | mean、p95、max、histogram | 某些样本权重极大；weighted loss spike | sample weight bug、preference/RL 数据权重错误 | 记录 weighted/unweighted loss；限制极端权重前先查根因 | 根因定位 |
| packed boundary legality | cross-sample attention ratio、boundary mask check | packed 样本互相 attend；label 跨样本 | packing mask bug、position id reset 错 | 构造 toy packed batch；检查 attention mask | 根因定位 |
| position id | min、max、range、reset pattern | position id 不递增、越界、跨样本未 reset | RoPE/packing/long-context bug | 记录 position id 与 sample boundary | 根因定位 |
| input token validity | token id min/max、OOV、vocab overflow | token id 越界或特殊 id 错 | tokenizer/model vocab 不一致 | 校验 tokenizer hash、vocab size、special ids | 根因定位 |


典型判断：

| 指标组合 | 优先判断 |
| --- | --- |
| 同一 batch replay 稳定 spike | 数据、mask、label、sequence 或 precision 是重点 |
| 去掉 top-loss sequence 后不 spike | 样本级数据问题概率高 |
| unpacked 正常，packed 异常 | packing boundary、mask、position id 问题 |
| 长序列分桶异常，短序列正常 | long-context、RoPE、attention score、memory precision |
| valid label ratio 极低但 loss 很高 | loss reduction 或 ignore_index 配置问题 |
| 特殊 token loss 极端 | tokenizer/template/label shift 问题 |

### 8.11 distributed rank 级别指标
分布式训练中，很多稳定性问题不是模型算法本身导致的，而是 rank 间状态不一致、通信错误、数据分片异常、checkpoint shard 错误或 loss scaler 不同步。rank 级别监控必须能回答：**所有 rank 是否在训练同一个模型、同一个 step、同一种数值状态。**

| 监控对象 | 推荐统计量 | 异常信号 | 可能根因 | 推荐处理方式 | 更适合用于 |
| --- | --- | --- | --- | --- | --- |
| per-rank loss | mean、p95、max、rank histogram | 某 rank loss 明显偏离；某 rank 先 NaN | rank 数据异常、loss reduction、mask bug | 保存该 rank batch；rank-wise replay | 根因定位 |
| per-rank grad norm | pre-allreduce norm、post-allreduce norm | allreduce 前某 rank grad 极大；post 后全局被污染 | rank-local bad batch、数据不均、同步错误 | 记录 pre/post allreduce；隔离异常 rank batch | 根因定位 |
| per-rank update norm | norm、U/W ratio | 某 rank update 与其他 rank 不一致 | optimizer shard、ZeRO/FSDP state、通信错误 | rank-wise state checksum；full-state 对比 | 根因定位 |
| per-rank overflow / found_inf | count、step id、loss scale | 只有某些 rank overflow；loss scaler 不同步 | fp16 local overflow、scaler sync bug | 同步 found_inf；定位 local first non-finite | 根因定位 |
| per-rank loss scale | value、trend | rank 间 loss scale 不一致 | AMP scaler 状态错误 | 强制同步或修 scaler state | 根因定位 |
| parameter checksum | checksum、hash、norm by rank | rank 间参数不一致 | DDP/FSDP/ZeRO 同步或 checkpoint 错 | 定期 checksum；异常时停止训练 | 根因定位 |
| optimizer state checksum | m/v checksum、finite ratio | rank 间 optimizer state 不一致 | state shard load/save、reshard 错 | 校验 shard metadata；full checkpoint round-trip | 根因定位 |
| batch shape by rank | seq length、tokens、microbatch size | rank 间 batch shape 差异异常 | sampler、packing、drop_last、dynamic batching | 记录 rank-wise batch metadata | 根因定位 |
| communication health | allreduce time、bucket time、timeout、retry | 某 step 通信时间异常，随后 loss 异常 | 通信错误、straggler、rank hang | 关联 step time 与 rank stats | 辅助定位 |
| gradient accumulation state | microstep id、no_sync 状态、accum count | rank 间 accumulation step 不一致 | no_sync 使用错误、resume microstep 错 | 校验 microstep 与 optimizer step | 根因定位 |


判断标准：

| 现象 | 优先排查方向 |
| --- | --- |
| 单卡稳定，多卡不稳定 | DDP/FSDP/ZeRO、allreduce、rank 数据、accumulation |
| 某 rank loss/grad 先异常 | rank-local batch 或 shard state |
| allreduce 前差异极大，allreduce 后全局异常 | 单 rank 污染全局梯度 |
| checkpoint resume 后 rank checksum 不一致 | shard load/save、reshard、state dict 映射 |
| loss scaler rank 间不一致 | AMP scaler 同步或 found_inf 聚合问题 |

### 8.12 MoE expert 级别指标
MoE 模型的稳定性监控必须同时覆盖 router、expert load、capacity、drop tokens、expert gradient/update。只看全局 loss 会漏掉 expert collapse、router collapse、局部 expert over-training 等问题。

| 监控对象 | 推荐统计量 | 异常信号 | 可能根因 | 推荐处理方式 | 更适合用于 |
| --- | --- | --- | --- | --- | --- |
| expert load | tokens/expert、mean、max、min、CV、max/mean | 少数 expert 过载；大量 expert 低负载或空载 | router collapse、aux loss 不足、capacity 设置 | 调 aux loss、router temperature、capacity；检查 routing logits | 根因定位 |
| router logits | RMS、p95、p99、max、histogram | router logits 极端；分布重尾 | router scale、LR、precision、z-loss 不足 | 监控 z-loss；考虑 router logits 正则或精度提升 | 定位 + 保护触发 |
| router entropy | mean、min、p5、histogram | entropy collapse；top expert 过于固定 | routing 过尖、logits 过大、temperature 不当 | 调 router temperature/z-loss/aux loss | 根因定位 |
| routing probability top-k | top-1 prob、top-k mass、margin | top-1 prob 接近 1；top-2 margin 过大 | expert collapse、routing 饱和 | 检查 load balance 与 token drop | 根因定位 |
| dropped tokens | drop ratio、by expert drop、by source drop | drop ratio 上升；某些 expert capacity 满 | capacity factor 低、expert load 不均 | 调 capacity、aux loss、batch routing | 根因定位 |
| capacity usage | usage ratio、overflow count | 长期接近 capacity 上限 | capacity 不足或 router 不均 | 调 capacity factor；检查 expert load CV | 根因定位 |
| expert activation norm | per-expert RMS、p99、max | 某 expert activation outlier | 专家过载、局部数据分布、expert 参数异常 | per-expert hook；按 source/token 分析 | 根因定位 |
| expert grad norm | per-expert norm、p95、max | 某 expert gradient 爆；低负载 expert 梯度稀疏异常 | load imbalance、expert-specific bad data、optimizer state | expert-wise grad clipping 只作保护，优先查 routing | 定位 + 保护触发 |
| expert update-to-weight ratio | per-expert U/W、p95、max | 某 expert update 远高于其他 expert | expert load、optimizer state、LR group | 检查 expert optimizer state 与 token load | 根因定位 |
| auxiliary load balancing loss | value、trend、ratio to main loss | aux loss 过大主导训练；或过小无法均衡 | aux weight 不合理 | 监控 aux/main loss ratio；调权重 | 根因定位 |
| router z-loss | value、trend | z-loss 高说明 router logits 尺度大；过强可能压制 routing | router logits 爆、z-loss 权重问题 | 与 router entropy/load 联合判断 | 定位 + 保护触发 |
| expert parallel rank stats | expert load by rank、expert state checksum | 某些 expert/rank 状态异常 | expert parallel 通信、shard load 错 | rank-expert 二维监控 | 根因定位 |


典型判断：

| 指标组合 | 优先判断 |
| --- | --- |
| router entropy 低，expert load CV 高 | router collapse 或 aux/z-loss 不足 |
| expert load 高且 drop ratio 高 | capacity factor 或 routing imbalance |
| expert grad 爆但 expert load 不高 | expert 参数/state 或该 expert 接收的数据异常 |
| aux loss 占比过高，main loss 下降变慢 | load balancing 目标过强，可能损害专家专化 |
| router logits 大但 load 尚可 | 可能处于 collapse 前期，应监控 z-loss 和 entropy trend |

### 8.13 指标之间的联动判断
单个指标很少足以判断根因。推荐使用“指标组合”进行分流。

| 指标组合 | 更可能根因 | 推荐下一步 |
| --- | --- | --- |
| loss spike + per-sequence loss top-k 极端 + replay bad batch 复现 | 数据、label、mask、sample weight | 保存 top-k 样本，逐样本 replay，检查 label/mask |
| loss spike + logits max 极端 + final hidden RMS 正常 | lm head、vocab、tokenizer、label shift | 检查 lm_head row norm、top logit token、special token |
| loss spike + final hidden RMS 极端 + layer-wise RMS 上升 | residual stream / attention / MLP 尺度问题 | 定位第一个 RMS 异常层 |
| grad norm 大 + activation 先异常 | forward signal explosion | 查 layer residual、attention score、MLP activation |
| grad norm 大 + activation 正常 + loss 极端 | loss/label/logits 或 objective 问题 | 查 per-token loss、label shift、loss reduction |
| grad norm 正常 + update norm 大 | optimizer state 或 LR group | 查 Adam m/v、epsilon、scheduler、param group |
| update-to-weight ratio 高 + weight norm 小 | 参数尺度或初始化问题 | 查初始化、weight decay、LR group |
| overflow count 高 + fp32 replay 正常 | precision / kernel 问题 | bf16/fp32、stable softmax、关闭 fused kernel |
| single-rank 正常 + multi-rank 异常 | 分布式一致性问题 | rank-wise loss/grad/checksum、ZeRO/FSDP state |
| attention entropy collapse + QK score p99 高 | attention score 尺度问题 | QK Norm、temperature、RoPE/mask 检查 |
| attention entropy collapse + mask legality failed | mask / packing bug | 构造 toy mask 测试，修 mask |
| MoE load CV 高 + router entropy 低 | router collapse | 调 aux/z-loss/temperature/capacity |
| clip rate 高 + valid loss 变差 | clip 掩盖问题或干扰优化 | 记录 clip 前后方向 cosine，回到根因定位 |

### 8.14 异常阈值与告警策略
稳定性阈值不宜只使用固定绝对值。不同模型规模、深度、dtype、batch size、context length、optimizer、数据混合都会改变正常范围。更稳健的做法是组合使用**历史基线、相邻层对比、rank 间对比、分布尾部、触发频率**。

| 告警方式 | 适用指标 | 判断标准 |
| --- | --- | --- |
| 绝对告警 | NaN、Inf、overflow、illegal attention、token id 越界 | 一旦出现即告警 |
| 历史基线告警 | loss、grad norm、update norm、logits RMS、residual RMS | 超过 warmup 后稳定窗口 p99 的若干倍 |
| 相邻层告警 | layer RMS、layer grad norm、U/W | 某层明显高于前后层或同类层 |
| rank 间告警 | per-rank loss、grad、overflow、checksum | 某 rank 偏离 rank median |
| 频率告警 | clip rate、overflow rate、skip step rate | 连续 N step 高于阈值，而非单次触发 |
| 分布形状告警 | histogram、p99/median、kurtosis | p99 与 median 脱钩，重尾加剧 |
| 方向告警 | update cosine、clip 前后 cosine | clip 后方向变化过大 |


推荐设置三档告警：

| 告警等级 | 触发条件 | 操作 |
| --- | --- | --- |
| warning | 指标偏离历史区间但无 NaN/Inf，loss 未持续恶化 | 加密日志采样，开启相关层级监控 |
| critical | loss spike、grad/update 大幅异常、overflow 连续出现、clip rate 升高 | 保存 checkpoint 和 batch，触发 replay，暂停或降低 LR |
| stop-the-line | NaN/Inf 进入参数或 optimizer state、rank checksum 不一致、非法 attention 确认 | 停止训练，回滚到安全 checkpoint，做根因修复 |


需要特别注意：  
**clip rate、overflow count、skip step count 不是“越低越好”的普通指标，而是保护机制是否被滥用的信号。** 偶发触发可以接受；长期高频触发说明训练过程已经依赖保护机制维持表面稳定，应回到 A 类根因定位。

### 8.15 监控指标的最小集合与增强集合
在大规模预训练中，完整记录所有 histogram 和 per-head/per-expert 指标可能成本较高。可以采用“最小常驻集合 + 异常触发增强集合”的策略。

#### 8.15.1 最小常驻集合
| 层级 | 最小常驻指标 |
| --- | --- |
| global | train loss、valid loss、LR、tokens/update、global grad norm、global update norm、update-to-weight ratio、loss scale、overflow count、clip rate |
| layer | layer residual RMS、layer grad norm、layer update-to-weight ratio、non-finite count |
| attention | attention score p99、attention entropy、max attention probability、mask legality sample check |
| MLP | MLP output RMS、activation p99、MLP grad norm |
| embedding / lm head | logits max/RMS/entropy、lm_head row norm p99、special token loss |
| optimizer | Adam m/v norm、effective update norm、state finite ratio |
| data | sequence length p99、per-sequence loss top-k、valid label ratio、special token ratio、source mix |
| distributed | per-rank loss、pre/post allreduce grad norm、loss scale、param checksum sample |
| MoE | expert load CV、dropped token ratio、router entropy、router logits max、expert grad norm |


#### 8.15.2 异常触发增强集合
当出现 loss spike、NaN/Inf、overflow、clip rate 升高、rank divergence、MoE collapse 时，临时开启：

| 异常 | 增强监控 |
| --- | --- |
| loss spike | per-sample/per-token loss、top-k bad samples、logits top-k、label/mask dump |
| grad explosion | full layer-wise backward hook、grad histogram、clip 前后 cosine |
| activation explosion | full forward hook、residual delta ratio、attention/MLP 分解 |
| logits explosion | final hidden RMS、lm_head row norm、top-token frequency |
| attention collapse | per-head Q/K norm、score histogram、entropy histogram、mask legality |
| optimizer update 异常 | Adam m/v histogram、effective update、decay/update 分解 |
| NaN/Inf | first non-finite tensor tracing、fp32 replay、kernel fallback |
| distributed divergence | per-rank batch dump、checksum、pre/post allreduce stats |
| MoE instability | per-expert token ids、router logits histogram、expert load by source/rank |

### 8.16 典型监控案例
#### 案例 1：loss spike 由异常 batch 触发
| 观察 | 指标 |
| --- | --- |
| global loss 在某 step spike | training loss p99、Δloss/step |
| global grad norm 随后变大 | global grad norm |
| 同 batch replay 可复现 | bad batch sample ids |
| per-sequence loss top-k 指向少数样本 | per-sequence loss top-k |
| 这些样本 valid label ratio 异常低 | label ignore ratio、valid label ratio |


判断：更像数据/label/mask 问题，而不是纯 LR 问题。  
处理：逐样本 replay，检查 label shift、ignore_index、template 和 sample weight。若修复后同 batch 不再 spike，属于 A 类根因修复。grad clipping 可以在修复前作为短期保险，但不能作为最终解决方案。

---

#### 案例 2：grad norm 正常但 update-to-weight ratio 异常
| 观察 | 指标 |
| --- | --- |
| global grad norm 没有明显 spike | global grad norm |
| global update norm 突然变大 | update norm |
| 某些层 U/W 明显高于其他层 | layer-wise update-to-weight ratio |
| Adam v 在这些层异常小 | Adam v min/mean |
| resume 后 optimizer step 不连续 | optimizer step、scheduler step |


判断：更像 optimizer state / resume / epsilon 问题，而不是梯度本身爆炸。  
处理：检查 optimizer state 加载、ZeRO/FSDP shard 映射、Adam step、scheduler step。必要时从正确 checkpoint 回滚。update clipping 只能作为临时保护。

---

#### 案例 3：attention entropy collapse 先于 loss 恶化
| 观察 | 指标 |
| --- | --- |
| 某些 head entropy 持续下降 | per-head attention entropy |
| QK score p99 持续上升 | attention score p99 |
| Q/K norm 同步上升 | Q norm、K norm |
| long sequence batch 更明显 | length-bucketed attention stats |
| mask legality 正常 | illegal attention ratio = 0 |


判断：更像 attention score 尺度 / QK norm / RoPE scaling / temperature 问题，而不是 mask bug。  
处理：检查 QK score scale、RoPE scaling、attention temperature；可进一步评估 QK Norm 或 logit softcap。若 hard clamp score 后 loss 稳定但 entropy 仍异常，说明只是 B 类止血。

---

#### 案例 4：MoE expert collapse
| 观察 | 指标 |
| --- | --- |
| 少数 expert 接收绝大多数 tokens | expert load CV、max/mean |
| router entropy 下降 | router entropy |
| dropped tokens 上升 | drop ratio |
| 过载 expert grad norm 大 | expert grad norm |
| aux loss 占比很低或 z-loss 异常 | aux/main ratio、z-loss |


判断：更像 router/expert load 机制问题。  
处理：调 router z-loss、aux load balancing loss、capacity factor、router precision 或 temperature。只对过载 expert 做 grad clipping 可以防止局部爆炸，但不能解决 routing collapse。

### 8.17 监控指标与 A / B / C 类方法的关系
监控指标不仅用于报警，也用于判断应该走 A、B 还是 C 类路径。

| 指标结论 | 更适合的处理路径 |
| --- | --- |
| 可以定位到数据、mask、label、LR、optimizer state、precision、distributed state 的具体错误 | A 类：根因修复 |
| 只能观察到数值幅度失控，但暂时找不到机制 | B 类：clip / clamp / rescale 临时保护，同时继续定位 |
| 指标显示长期尺度或训练动力学不适配，例如深层 residual RMS、attention entropy、U/W ratio 系统性异常 | C 类：考虑架构、参数化、normalization、residual scaling、warmup、batch/LR recipe |
| 保护机制高频触发且质量变差 | 回到 A 类定位，检查保护是否掩盖根因 |
| 保护机制低频触发且不影响 valid loss / downstream | 可作为保险丝保留，但应持续记录触发率 |


最终，稳定性监控的合格标准不是“日志很多”，而是满足以下条件：

| 标准 | 具体要求 |
| --- | --- |
| 可发现 | loss spike、NaN/Inf、overflow、rank divergence、expert collapse 能在早期被捕捉 |
| 可定位 | 能从 global 下钻到 layer/head/expert/rank/sample/token |
| 可判断 | 能区分数据、代码、LR、optimizer、架构、precision、distributed、MoE 问题 |
| 可触发 | 能安全触发 clip、skip step、rollback、bad batch quarantine、enhanced logging |
| 可复盘 | 异常 step 的 checkpoint、batch、rank stats、tensor stats 可重放 |
| 可约束副作用 | 能监控 clip rate、softcap saturation、update direction cosine、valid loss 变化 |
| 可闭环 | 修复后能证明指标回到稳定区间，而不是只把异常压到不可见 |
